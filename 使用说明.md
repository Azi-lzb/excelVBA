# VBA 报表工具 — 使用说明

本工作簿内置一组用于报表处理、机构数据标准化、地区核对与工作簿比对的 VBA 工具。打开工作簿并启用宏后，Excel 菜单栏会自动出现 **「报表工具(&S)」**，所有主要功能均通过该菜单调用，无需到 VBA 编辑器操作。

---

## 一、环境与前置设置

- **工作簿格式**：保存为 `.xlsm`，打开时选择「启用宏」。
- **信任设置（必配）**：  
  **文件 → 选项 → 信任中心 → 信任中心设置 → 宏设置** 中勾选 **「信任对 VBA 工程对象模型的访问」**。
- **推荐的数据工作表（按需准备，表名用中文即可）**：
  - **机构映射表**：用于机构名称标准化、识别外资行。
  - **工作表提取**：配置要从哪些模板文件里提取哪些工作表/区域。
  - **表格比对**：配置两个工作簿之间的对比规则。
  - **config_rename**：用于 3.3 批量重命名文件（仲恺报表银行命名格式：A/B/D/E 列简称·全称·代码，G/H 列键值如数据日期、报表名称）。
- **VBA 模块文件（.bas）编码说明**：  
  `VBA_Export\Modules` 下的 `.bas` 文件需以 **ANSI**（Windows 下即 GBK）保存，以便在 Excel VBA 编辑器中正确显示中文。生成或修改 .bas/.cls 后，请对相应文件运行 `python scripts/txt_save_as_ansi.py <文件路径>`（将文件用文本方式打开后另存为 ANSI）。  
  若在 **Cursor / VS Code** 中打开时出现乱码或问号：请点击编辑器右下角的编码（如「UTF-8」）→ 选择 **「通过编码重新打开」** → 选择 **「Chinese (Simplified, GB2312)」** 或 **GBK**，即可正常显示。

## 二、「报表工具(&S)」菜单功能

### 2.1 拆分村镇银行（三币种、外发用）

- **菜单路径**：`报表工具 → 2. 一二批功能 → 2.1 拆分村镇银行（三币种、外发用）`
- **底层过程**：`批量填入村镇银行计算结果`
- **用途**：从「本期」与「去年同期」两份汇总文件中，计算村镇银行本外币/人民币/外汇的存款和贷款数据及同比；在一批「分机构」报表中，自动拆分「村镇银行」为两行（博罗长江村镇银行、惠东惠民村镇银行），并填入计算结果。
- **使用步骤**：
  1. 点击菜单 1.1。
  2. 按提示选择 **本期文件**（包含村镇银行汇总数据）。
  3. 再按提示选择 **去年同期文件**。
  4. 然后多选需要填数的 **分机构文件**（名称中一般带"分机构"）。
  5. 程序会自动处理每个文件中相关工作表，完成拆分与填数。

### 2.2 将机构名称标准化（三币种、外发用）

- **菜单路径**：`报表工具 → 2. 一二批功能 → 2.2 将机构名称标准化（三币种、外发用）`
- **底层过程**：`批量修改分机构文件`
- **依赖工作表**：`机构映射表`（A 列：原机构名称；B 列：标准化后的机构名称）。
- **用途**：对一批「分机构」报表中机构列（如 A 列），按映射表替换为统一命名，方便汇总与比对。
- **使用步骤**：
  1. 在当前工作簿中维护好 `机构映射表`（至少 A、B 两列）。
  2. 点击菜单 1.2，选中一批分机构文件。
  3. 程序会自动逐个文件打开、根据映射规则替换机构名称并保存。

### 2.6 提取工作表（本外币、专项二批次）

### 3. 工作簿操作

- **菜单路径**：`报表工具 → 2. 一二批功能 → 2.6 提取工作表（本外币、专项二批次）`
- **底层过程**：`批量提取工作表数据`
- **依赖工作表**：`工作表提取`
- **用途**：根据配置行，从多份源文件中按规则筛选/复制工作表（或局部区域），重新组合生成新的工作簿。
- **配置说明（在 `工作表提取` 表中）**：
  - A：表A-币种关键字
  - B：表A-地区关键字
  - C：表A-机构关键字
  - D：表A-类型关键字
  - E：表A-名称关键字（A–E 综合决定要找哪些 sheet）
  - F：是否禁用（1=本行规则不生效）
  - G：是否整表提取（1=整表复制；0=只复制指定区域）
  - H：提取行，如 `2:100`（留空则不限制行）
  - I：提取列：支持 **单列/范围/组合**，如 `A:Z`、`B,BO,BQ,BP`、`B,BO:BP`（留空则不限制列）
  - J：输出文件名（同名多行可汇总到同一个结果工作簿）
- **使用步骤**：
  1. 在当前工作簿填写好 `工作表提取` 每一行规则。
  2. 点击菜单 2.6，多选作为 **源数据** 的 Excel 文件。
  3. 程序按配置逐行执行：找到匹配的 sheet → 按设定范围复制 → 按输出文件名分类保存。

### 1.6 模板与源数据表格校验

- **菜单路径**：`报表工具 → 1. 文件选择 → 1.6 模板与源数据表格校验`
- **底层过程**：`模板与源数据表格校验`（模块 `功能1dot6模板与源数据表格校验`）
- **用途**：根据执行面板 **A2 模板路径** 与 **B 列源文件列表**（B5 起），对每个源文件做两项校验：  
  - **表格数量校验**：检查源文件的工作表数量及名称是否与模板一致，结果写入 **D 列（表头 D4=表格数量校验）**。  
  - **表格样式校验**：根据 `config` 表配置，比对指定区域内的单元格内容是否一致。校验逻辑与 **3.2.2 按批注汇总** 保持同步。
  
- **校验逻辑（受 config 控制）**：
  - **强制按模板** 为「是」：程序仅提取模板中名为 **「模板」** 的工作表内的批注区域，并以此为基准，对源文件中的 **所有工作表** 同一位置进行比对。
  - **强制按模板** 为「否」：
    1. 寻找源文件与模板文件中的 **同名工作表** 进行比对。
    2. 若源文件中的工作表在模板中无同名匹配，则使用模板中名为 **「模板」** 的工作表（若存在）作为兜底基准进行比对。
- **批注识别规则**：
  - 在模板中使用 `行区域1` / `行区域#1` 或 `列区域1` / `列区域#1`（支持多组 N/#N）围成矩形。
  - `行区域` 与 `列区域` 分别按各自围成的矩形独立校验，二者不会因为编号相同而合并成更大的区域。
  - 程序将对这些矩形区域的并集逐格进行 `Trim` 后的文本比对；若行区域与列区域有重叠单元格，仅校验一次。
- **配置说明（config 表）**：  
  - 键 `1.6 模板与源数据表格校验`：
    - **行区域**：值填「是/1/true」则执行行区域批注矩形校验。
    - **列区域**：值填「是/1/true」则执行列区域批注矩形校验。
  - 键 `2.2.2 按批注汇总`：
    - **强制按模板**：控制上述校验逻辑的分支选择。
- **输出结果**：
  - **D 列**：数量不一致时记录源文件与模板文件的 Sheet 名清单。
  - **E 列**：样式不一致时记录具体位置，格式为：`源表名:单元格与模板文件 模板表名:单元格不一致`，多条差异用分号分隔。
- **示例**：
  - 若 `列区域` 为 `A2:C3`，`行区域` 为 `A3:A5`，则实际只校验 `A2:C3` 与 `A3:A5` 这两个区域；`B4:C5` 不参与校验。
- **使用步骤**：
  1. 在执行面板 A2 填入模板，B5 起填入源文件。
  2. 在 `config` 表确认校验开关（行区域/列区域）及模式（强制按模板）。
  3. 点击菜单 1.6，完成后根据弹窗提示查看 D、E 列的详细报告。

### 1.7 注入校验区域

- **菜单路径**：`报表工具 → 1. 文件选择 → 1.7 注入校验区域`
- **底层过程**：`注入校验区域`（模块 `功能1dot7注入校验区域与汇总校验结果`）
- **用途**：将模板工作簿中各**同名工作表**里，用批注围成的「校验区域」矩形（可多组）内的 **批注 / 单元格值 / 公式**，复制到每个源文件的同名工作表的**相同单元格位置**，并保存源文件。用于把同一套校验公式批量应用到多张表，减少人工逐表复制。
- **模板批注关键字**（两个批注围成矩形）：  
  - **本地校验区域1** 与 **本地校验区域#1**（可有 2/#2、3/#3…）  
  - **跨表校验区域1** 与 **跨表校验区域#1**（可多组）
- **使用步骤**：
  1. 在执行面板 A2 填好模板路径，B5 起填好源文件路径列表。
  2. 在模板各工作表需要注入的区域，用上述关键字批注标出矩形范围（区域内每个单元格可以有自己的批注说明）。
  3. 点击 1.7：程序逐个打开源文件（可写），将模板对应区域的**公式/值/批注**注入到源文件同名 sheet 并保存。

---

### 1.8 汇总校验结果

- **菜单路径**：`报表工具 → 1. 文件选择 → 1.8 汇总校验结果`
- **底层过程**：`汇总校验结果`（模块 `功能1dot7注入校验区域与汇总校验结果`）
- **用途**：读取源文件（只读）中被 1.7 注入的校验区域矩形内的单元格值（包含公式计算结果），提取包含关键字 **错 / 校验失败 / 硬性 / 软性 / 警告** 的单元格，汇总到本工具工作簿的工作表 **「校验结果」**（没有则自动新建）。
- **提取规则**：
  - **命中关键字**：单元格值包含「错」或「校验失败」或「硬性」或「软性」或「警告」即记录一行。
  - **错误类型与预留字段**：若单元格值形如 `错误类型|预留1|预留2|预留3|预留4`，则按 `|` 拆分后分别写入「错误类型」「预留字段1~4」。
  - **批注描述**：提取该错误单元格自身的批注文本写入「错误单元格的批注描述」。
- **结果表字段（表头）**：模板工作簿名、模板工作表名、源文件工作簿名、源文件工作表名、行序号、列序号、行名称、错误类型、预留字段1、预留字段2、预留字段3、预留字段4、错误单元格的批注描述。
- **使用步骤**：
  1. 先运行 1.7 注入校验区域（让源文件具备校验公式与批注说明）。
  2. 点击 1.8：程序逐个打开源文件，扫描模板同名 sheet 的校验区域矩形，输出汇总表。

### 3.1 做一些表格之间的对比（通用）

- **菜单路径**：`报表工具 → 3. 工作簿操作 → 3.1 做一些表格之间的对比`
- **底层过程**：`执行表格比对`
- **依赖工作表**：`表格比对`
- **用途**：比较两个工作簿中相应工作表的数值/文本差异，支持全表逐格比对或按指定单元格位置比对。
- **配置思路（在 `表格比对`）**：
  - 第 1 行表头用于说明；从第 2 行起每行一条规则。
  - A–E：工作簿 A 的 sheet 名关键字（全部包含才算匹配）。
  - F–J：工作簿 B 的 sheet 名关键字。
  - K：是否执行（1=执行本行规则）。
  - L：是否全量比对（1=整张表逐格比对；0=按位置比对）。
  - M：表 A 的单元格/区域（按模块实现约定）。
  - N：表 B 的单元格/区域。
- **使用步骤**：
  1. 按上述格式填好 `表格比对`。
  2. 点击菜单 2.1，依次选择需要比对的 **工作簿 A** 和 **工作簿 B**。
  3. 程序打开两个工作簿，根据每行配置执行比对，输出到新建工作簿的「比对结果」工作表（包含单元格地址、A/B 值、是否一致等）。

### 3.2 汇总

菜单路径：`报表工具 → 3. 工作簿操作 → 3.2 汇总`，下含两个子功能。

#### 3.2.1 按使用区域汇总

- **菜单路径**：`报表工具 → 3. 工作簿操作 → 3.2 汇总 → 3.2.1 按使用区域汇总`
- **底层过程**：`SummarizeByUsedRange`（模块 `SummarizeByUsedRange`）
- **用途**：批量选择 Excel/CSV/WPS 表格等文件，遍历每个工作簿下的所有工作表；若有数据则取该工作表的 UsedRange，将数据汇总到新建工作簿中。汇总表表头为：**工作簿**、**工作表**、**行**、**列1**…**列N**（列数以第一个遇到的 UsedRange 为准）。合并单元格取左上角值。
- **使用步骤**：
  1. （可选）在当前工作簿中维护「config」表，配置「跳过表头」等选项。

  5. 程序自动创建新工作簿、写入「数据汇总」表后，弹出「另存为」对话框。

- **config 配置项**：见下方「初始化 config 配置表」章节中 `按使用区域汇总` 行（菜单为 3.2.1）。

#### 3.2.2 汇总

- **菜单路径**：`报表工具 → 3. 工作簿操作 → 3.2 汇总 → 3.2.2 汇总`
- **底层过程**：`SummarizeBySheetThenSplit`（模块 `SummarizeBySheetThenSplit`）
- **用途**：从**执行面板**读取模板路径（A2）与源文件列表（B5 起），按模板批注汇总。支持两种模式（由 config **强制按模板** 控制）：
  - **未勾选强制按模板**：按 Sheet 名与模板中带批注的工作表一一对应，结果按表类型分多张工作表；可配置「参与/不参与的工作表」过滤。
  - **勾选强制按模板**：仅使用模板中的「模板」表，所有源 Sheet 都汇总到一张结果表「ALL」，与旧版「按模板汇总」行为一致。
- **分列**：config 中 **按批注汇总** 的**分列**非 0 时，汇总结束后对结果表指定列按分隔符拆成多列；**分列=0 或留空**则不分列。
- **配置**：config 键 **按批注汇总** 下可配置：**工作簿**、**工作表**、**set区**、**列区域**、**行号**、**跳过未匹配表**、**分列**、**不参与的工作表**、**参与的工作表**、**强制按模板**。分列格式：`列:分隔符`；多规则用分号分隔如 `A: ;B:_`。

#### 批注约定（通用模板汇总设计）

本工具通过**模板**定义"哪里是什么数据"，使用**特殊关键字**批注，避免与日常批注冲突；换表样只需换模板。

**一、单格字段：set(字段名)**  
在单元格批注中写 **`set(数据日期)`**、**`set(机构名称)`**、**`set(单位)`**，表示该格的值即该字段。  
**中文括号与英文括号等价**：`set（数据日期）` 与 `set(数据日期)` 视为相同。字段名可扩展，按批注出现顺序排列为结果列。源表与模板**版式一致**，按**相同单元格位置**取数。

**二、表头区域：列区域1 与 列区域#1**  
用两个批注 **`列区域1`** 和 **`列区域#1`** 标出两个单元格，二者围成的**矩形**即为表头区域。  
- 矩形为多行多列时，**每一列**对应结果表的一列；该列的表头 = 该列内所有单元格的值用 **下划线 `_`** 依次拼接。  
- 例：两格围成 A1:B2，则表头有两列：第 1 列表头 = `A1的值_A2的值`，第 2 列表头 = `B1的值_B2的值`。  
- 若矩形只有一行，则表头为各单元格的值。支持多组（列区域2/#2 等）。

**三、数据行区域：行区域1 与 行区域#1**  
用两个批注 **`行区域1`** 和 **`行区域#1`** 标出两个单元格，二者围成的区域确定**数据行范围**。  
- **取数区域** = 行区域（行范围）与 列区域（列范围）的**交叉**。  
- 例：列区域为 A1:B2（列 A、B），行区域为 A3:A4（第 3、4 行），则交叉为 A3:B4；共 2 行 2 列，依次输出 2 行数据。支持多组行区域。

**四、结果表结构**  
可通过 config 配置列开关（默认全开）：**工作簿**、**工作表**、**set 字段列**（如数据日期、机构名称、单位等）、**列区域列**（表头与数据）、**行号**（默认关）。无 set 的字段留空；列顺序与批注及列区域从左到右一致，每行数据为交叉区域内的一行，从上到下依次汇总。  
（**分列**在 3.2.2 的 config 中配置，分列=0 或留空即不分列。）

**五、同名表汇总与「模板」表**  
模板工作簿中**带上述关键字批注**的工作表名 = **表类型**。在执行面板中填写模板路径（A2）与源文件列表（B5 起），或通过「选择模板文件」「选择源文件」等按钮添加。  
- 源表中**与模板同名**的工作表用该模板取数。  
- 若模板中存在名为 **「模板」** 的工作表，则**未匹配到同名**的源表也用「模板」解析。  
- 结果工作簿中：**同一表类型**对应**一张工作表**，表名与类型名相同（**强制按模板**时仅一张表「ALL」）。  

**六、使用流程**
1. 制作**模板工作簿**：复制一份目标表样，在对应单元格上加上述**特殊关键字**批注（set、列区域1/#1、行区域1/#1）。强制按模板时模板中须有名为「模板」的工作表。  
2. 运行菜单 **3.2.2 汇总**：若执行面板未填模板路径会先弹窗选择模板；源文件从执行面板 B 列读取，可事先用「选择源文件」等添加。  
3. 程序按模板解析每个源表，汇总后弹出「另存为」对话框。  

这样**换表样只需换模板**，且批注关键字特殊，不易与日常批注混淆。

- **config 配置项**：见下方「初始化 config 配置表」章节中 `按批注汇总` 相关行（菜单为 3.2.2）。

### 3.5 追加列（按批注）

- **菜单路径**：`报表工具 → 3. 工作簿操作 → 3.5 追加列（按批注）`
- **底层过程**：`追加列按批注`（模块 `追加列按批注`）
- **用途**：根据**模板文件**各工作表**第一行表头**的批注，找出批注文本包含「追加列」的列，形成 **{工作表名：列字母}** 字典。**关键字仅匹配工作簿名**（不匹配工作表名）：源文件**名**包含某关键字时，在该源工作簿内**遍历所有工作表**，若某工作表名在字典中则取该表对应列；在**外部文件**中按该关键字找到同名工作表（无则新建），并往该表最右列追加，表头命名为 **工作簿名_工作表名_列字母列**（如 `Report.xlsm_Sheet1_A列`）。
- **执行面板约定**：**A2** = 模板文件完整路径，**B2** = 外部文件完整路径，**B5** 起 = 源文件路径列表（与 5.4/5.5 同列）。
- **配置说明（config 表）**：
  - **键**：`追加列（按批注）`
  - **键名**：`关键字` → 值：分号分隔的文本，如 **sheet1;sheet2**。仅用于匹配**源工作簿文件名**；同时该关键字作为**外部文件中工作表的名称**，列将追加到外部文件中与该关键字同名的表。
  - **键名**：`追加列批注` → 值：模板表头批注需包含的文本，默认 **追加列**（**必填**，不可省略）。若表头某单元格批注包含该文本，则该单元格所在列参与字典（工作表名 → 列字母）。
- **使用步骤**：
  1. 在模板工作簿中，为每个要参与的工作表在**第一行**对应列的表头单元格添加批注，批注内容包含「追加列」（或 config 中配置的文本），程序会据此形成「工作表名 → 列字母」字典。
  2. 执行面板 A2 填模板路径、B2 填外部文件路径、B5 起填源文件路径。
  3. config 中配置「追加列（按批注）」-「关键字」为如 `sheet1;sheet2`。
  4. 点击菜单 3.5；若 A2/B2 为空会先弹窗选择模板与外部文件。程序打开模板建字典、打开外部文件，再逐个打开源文件：若**源文件名**包含某关键字，则遍历该源工作簿**所有工作表**，对字典中存在的表名取对应列，追加到**外部文件中与该关键字同名的工作表**最右列，表头为 工作簿名_工作表名_列字母列。
- **依赖与提醒**：需 **Microsoft Scripting Runtime** 或通过 CreateObject("Scripting.Dictionary") 可用。模板/外部/源文件均为 Excel 工作簿。

### 4.3 分列（数字/符号/中文）

- **菜单路径**：`报表工具 → 4. 工作表操作 → 4.3 分列（数字/符号/中文）`
- **底层过程**：`分列数字符号中文`（模块 `分列数字符号中文`）
- **用途**：对**选中的一列**按字符类型拆成**三列**：数字（0–9 连续）、符号（如 `* - / +` 等）、中文（含字母、CJK 等其余字符）。例如 `0212*林木育苗` → 三列分别为 `0212`、`*`、`林木育苗`；`1563 *-/+ 阿斯顿` → `1563`、` *-/+ `、`阿斯顿`。程序会在选中列右侧**插入两列**，共三列存放结果。
- **配置说明（config 表）**：
  - **键**：`分列数字符号中文`
  - **键名**：`分列顺序` → 值：三列依次对应的类型，**分号分隔**，如 `数字;符号;中文`（默认）。可调顺序，例如 `中文;数字;符号` 表示第一列放中文段、第二列放数字段、第三列放符号段。
  - **键名**：`符号集`（可选）→ 值：视为「符号」的字符集合，不含分号。不配置时使用默认符号集（含空格、`*\-/+,.;:!?@#$%^&()[]` 等）。
- **使用步骤**：
  1. 在 config 中可配置「分列顺序」（如 `数字;符号;中文`）及可选「符号集」。
  2. 在工作表中**选中要分列的那一列**（或该列中任意单元格）。
  3. 点击菜单 4.3；程序在该列右侧插入两列，并将每行单元格按数字/符号/中文拆成三列写入。

- **底层过程**：`批量转时序数据`（模块 `功能3dot14批量转时序`）
- **依赖工作表**：`时序提取规则`
- **用途**：按规则批量选择多个工作簿，定位匹配的工作表，将二维表按「行头路径 + 列头路径 + 数值」展开成长表，输出到**新建结果工作簿**中的 `时序提取结果`，便于后续 `SUMIFS`、核对或建中间数据池。
- **执行前提醒**：程序启动时会先弹框，要求用户先确认目标工作簿已完成表格校验且无错误。
- **规则表核心字段**：
  - `是否启用`、`规则名称`
  - `工作簿关键字`、`工作表关键字`
  - `行头列`、`列表头行`
  - `必含列头`、`必含行头`
  - `数据起始行/结束行`、`数据起始列/结束列`
  - `跳过关键字`
- **提取口径**：
  - 程序先按工作簿关键字命中文件，再按工作表关键字命中 sheet。
  - 行头路径、列头路径默认按文本拼接生成；同一张 sheet 内若出现**完全相同**的行头路径或列头路径，会自动编号为 `名称_1`、`名称_2`。
  - 自动编号仅在**同一张 sheet 内**生效，不跨 sheet 计数；未重复的路径保持原样。
  - 去重仍按业务口径执行：`源文件 + 工作表名 + 数据日期 + 行头路径 + 列头路径 + 数值`。
- **结果表字段（表头）**：
  - 执行时间、源文件、工作表名、规则名称、源文件修改时间、数据日期、日期来源、行头路径、列头路径、数值、单元格地址。
- **运行日志补充**：
  - 若发生自动编号，会写日志 `行头重命名` / `列头重命名`。
  - 日志会记录：工作簿、工作表、规则、原始值、调整后值、位置。
  - 多行列头时位置格式为：`列C 行2,3`；行头位置格式为：`R25`。
- **使用建议**：
  - 大表优先填写 `数据结束行/结束列`，避免把整张表都展开。
  - 对于校验区、说明区、第二块重复表，优先用 `数据结束行/列` 限定区域；`跳过关键字` 仅用于补充过滤。
  - 由于整块表展开后记录数会明显多于按批注汇总，适合做底层时序明细，不适合替代所有固定指标提取场景。

### 3.9 指标关系校验

- **菜单路径**：`报表工具 → 3. 工作簿操作 → 3.9 指标关系校验`
- **底层过程**：`执行指标关系校验`（模块 `功能1dot9指标关系校验`）
- **依赖工作表**：`指标校验规则`
- **用途**：按规则校验表单内指标之间的勾稽关系，如相等、求和、大于等于求和等；结果输出到**新建结果工作簿**中的 `指标校验结果`。
- **规则特点**：
  - 支持按工作表名关键字和表单编码共同命中 sheet，再去重后逐张执行。
  - 未命中任何工作表的规则会跳过，不写入结果，也不计入失败。
  - 适合维护「父项=子项之和」「含其中：时父项>=子项和」等规则。

#### 初始化 config 配置表（3.3）

- **宏名**：`初始化config`（菜单 6.1 或 Alt + F8 运行，或由其他模块在需要时调用）
- **行为**：
  1. 若当前工作簿中没有名为 `config` 的工作表，自动新建并写入表头（键 / 键名 / 值 / 备注）和全部默认配置。
  2. 若已有 `config` 表，逐条检查 **A列（键）+ B列（键名）** 是否已存在（不区分大小写）：  
     - 不存在 → 追加该配置行  
     - 已存在 → 跳过，不覆盖用户已修改的值  
  3. 运行结束后提示新增了多少条配置。
- **默认配置项一览**：

  | A 列（键） | B 列（键名） | C 列（默认值） | D 列（备注） |
  |---|---|---|---|
  | 按使用区域汇总 | 跳过表头 | 否 | 填 是/1/true 开启；留空或否则弹窗询问（菜单为 3.2.1） |
  | 按批注汇总 | 工作簿 | 1 | 结果中是否包含工作簿列（默认是）；菜单为 3.2.2 |
  | 按批注汇总 | 工作表 | 1 | 结果中是否包含工作表列（默认是） |
  | 按批注汇总 | set区 | 1 | 是否提取 set() 字段（默认是） |
  | 按批注汇总 | 列区域 | 1 | 是否提取列区域数据（默认是） |
  | 按批注汇总 | 行号 | 0 | 是否追加行号列（默认否） |
  | 按批注汇总 | 跳过未匹配表 | 0 | 源表无匹配模板时是否跳过（默认否） |
  | 按批注汇总 | 分列 | 0 | 0=不分列；A: =A列按空格；多规则用;分隔如 A: ;B:_ |
  | 按批注汇总 | 不参与的工作表 | （空） | 分号分隔，匹配到的表不参与汇总 |
  | 按批注汇总 | 参与的工作表 | （空） | 分号分隔，仅匹配到的表参与；与不参与同时配置时以此为准 |
  | 按批注汇总 | 强制按模板 | 0 | 1/是/true=仅用「模板」表、只生成一张表 ALL；0=按 Sheet 名与参与/不参与规则 |
  | 批量Excel格式转换 | 目的格式 | xlsx | 另存为格式：xls,xlsx,xlsm,csv,xlt,xltx,xltm,xlsb；结果存源目录下同名子文件夹（菜单为 3.4） |
  | 批量Word格式转换 | 目的格式 | docx | 另存为格式：doc 或 docx；结果存源目录下同名子文件夹（菜单为 3.6） |

> 后续新增功能若需要配置，只需在 `GetDefaultConfigs()` 函数中添加行，再运行一次 `初始化config` 即可自动补全。

### 1. 模块「遍历的一些功能」

- **遍历所有工作表**  
  宏名：`遍历所有工作表`。遍历当前工作簿所有工作表，在"立即窗口"输出表名、索引、A1 单元格内容。

  多选文件后，将各工作表的数据汇总到一个新工作簿的 `数据汇总` 表中（前两列自动记录来源工作簿名、工作表名；可选择是否跳过源表首行标题；程序结束时会提示保存路径）。

### 2. 模块「vbaSync」（代码同步，对应菜单 5.1 导入 VBA 等）

- **约定目录结构**：在当前工作簿同级目录下自动使用 `VBA_Export\` 文件夹：

  - `Classes\`：类模块、文档模块（`.cls`）

## 四、常见问题

- **菜单不出现或消失**  
  `ThisWorkbook` 的 `Workbook_Open` / `Workbook_Activate` 会重新挂载菜单。关闭后重新打开工作簿，或在 VBA 中运行 `报表工具` 过程即可。
- **报错「找不到工作表」或「配置表为空」**  
  检查当前工作簿中是否存在 `机构映射表` / `工作表提取` / `表格比对`，且列字段填写完整；配置表中用于拼接 sheet 名的关键字尽量简短稳定，避免多余空格。
- **运行较慢**  
  多数批处理已关闭屏幕刷新与事件；若源文件很多，建议分批处理或先用少量文件验证配置。
- **时序提取结果太大**  
  `3.8 批量转时序数据` 会把整块二维表展开成长表，数据量通常远大于按批注汇总。建议优先限制 `数据结束行/结束列`，并通过 `必含列头/必含行头`、`跳过关键字` 缩小范围。
- **时序提取中行头/列头被自动改名**  
  当同一张 sheet 内出现重复行头或列头时，程序会自动改成 `名称_1`、`名称_2` 并写运行日志。若你需要确认调整是否正确，请查看运行日志中的 `行头重命名`、`列头重命名` 记录。

如有新增菜单项或模块，可在本「使用说明」中按 **菜单路径 → 作用说明 → 依赖表 → 操作步骤** 的结构继续补充。
